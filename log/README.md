# Log 目录说明

## 目录用途
此目录用于存放项目的日志和说明文档，记录项目状态、变更历史和重要信息。

## 当前文件

### 1. 项目情况说明.md
- **文件**: `项目情况说明.md`
- **用途**: 详细记录项目的整体情况、结构、状态和历史
- **内容**:
  - 项目基本信息
  - 项目结构说明
  - 当前项目状态
  - 技术架构
  - Git提交历史
  - 已修复的GitHub Issues
  - 文件统计
  - 特殊说明
  - 访问链接
  - 维护说明

## 更新频率
- **项目情况说明.md**: 在项目有重大变更时更新
- **其他日志文件**: 根据需要创建

## 文件命名规范
- 使用中文或英文描述性名称
- 日期格式: YYYY-MM-DD
- 文件扩展名: .md (Markdown格式)

## 建议的日志文件
1. **变更日志.md** - 记录每次重要变更
2. **问题记录.md** - 记录遇到的问题和解决方案
3. **性能报告.md** - 记录性能测试结果
4. **用户反馈.md** - 记录用户反馈和建议

## 注意事项
- 日志文件应保持简洁明了
- 重要信息应突出显示
- 定期整理和归档旧日志
- 避免存储敏感信息
